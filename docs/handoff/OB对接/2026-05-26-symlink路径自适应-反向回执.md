### 事 #2(P2,新发现):QuickAdd data.json 双写陷阱(强烈建议升级 install.sh)

**问题**:install.sh Step 6 生成的 `.quickadd-choices.json` snippet 只覆盖 `choices[].macro.commands[].path`,但 QuickAdd 同一份 macro 在 `macros[]` 顶层数组**也存了一份 path**(因为 QuickAdd choice 可以引用 macros[] 里的 macro,所以双写)。

**实证**(2026-05-27):本次 OB CC Step 5 初版 Python 脚本只扫 `d.get("choices", [])`,**漏改 2 处 macros[] 里的 path**,grep 复检才发现(`scripts/feishukanban-ob-sync` 残留 2 处)。普通用户手动 merge .quickadd-choices.json 时**绝不会注意到**这个双写结构,迁移后 4 个 Cmd+P 命令里有 2 个会失效。

**建议升级方案**(任选):

**方案 A**(轻):install.sh Step 6 输出时加 warning:
```
⚠️ 注意:QuickAdd data.json 是双写结构 ——
choices[].macro.commands[].path 和 macros[].commands[].path 两处都要改。
本 snippet 只覆盖 choices[],如果你已有同名 macro,记得手动改 macros[] 里的 path。
```

**方案 B**(中):install.sh Step 6 改为**直接 Python 改写** `.obsidian/plugins/quickadd/data.json`(参考 OB CC 本次用的脚本),扫描 choices[] + macros[] 两处所有 path 字段做 prefix 替换,而不是输出 snippet 让用户手动 merge。优点:零手动错误 + idempotent;缺点:Python 依赖(install.sh 已经有 python3 检查,不算新依赖)。

**方案 C**(重):install.sh 不再管 QuickAdd,改为输出"运行后请跑 `python3 tools/install-quickadd.py`"这种独立工具,职责分离。

OB CC 推荐 **方案 B** —— 跟 v0.3.2 install.sh 已经"自动化 vault 端 90% 工作"的设计哲学一致。

### OB CC 本次用的 macros[] 漏改修复脚本(给独立 CC 参考)

```python
import json

PATH = "/Users/aim5/Documents/OB/.obsidian/plugins/quickadd/data.json"
OLD_PREFIX = "scripts/feishukanban-ob-sync/userscripts/"
NEW_PREFIX = "01 Project/00 进行中/06 小工具开发/feishukanban-ob-sync/userscripts/"

with open(PATH, encoding="utf-8") as f:
    d = json.load(f)

# 注意:扫两个并行数组!
for c in d.get("choices", []):
    if "macro" in c and c["macro"].get("commands"):
        for cmd in c["macro"]["commands"]:
            p = cmd.get("path", "")
            if p.startswith(OLD_PREFIX):
                cmd["path"] = NEW_PREFIX + p[len(OLD_PREFIX):]

for macro in d.get("macros", []):
    for cmd in macro.get("commands", []):
        p = cmd.get("path", "")
        if p.startswith(OLD_PREFIX):
            cmd["path"] = NEW_PREFIX + p[len(OLD_PREFIX):]

with open(PATH, "w", encoding="utf-8") as f:
    json.dump(d, f, ensure_ascii=False, indent=2)
```
