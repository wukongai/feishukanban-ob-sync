# 记录今日明细 userscript — 加「用时 / 完成度」两个字段(v1 → v2)

## 一句话需求

`quickadd-记录今日明细.js` 目前只采集「执行状态 + 一句描述(→ `review=`)」,用户希望再加 **⏱️ 用时(→ `act=`)** 和 **🎯 完成度(→ `done=`)** 两个**可选** prompt,产出标准执行明细行。

## 背景

- 执行明细子表是你 2026-05-29 加的 **v0.6.0** 功能(`sync.py` 的 `parse_execution_details` + `_DETAIL_KEY_ALIASES` + `push_execution_details`)。
- 当前 `quickadd-记录今日明细.js`(v1)只写 `- 日期 | 状态 | review=<描述>`。
- 用户日常想顺手记「今天这个 task 花了几小时 + 完成到什么程度」,手敲 `act=1.5 / done=标准完成` 太麻烦,要做成 inputPrompt / suggester。

## ⚠️ 边界说明(为什么走 handoff)

OB CC 一度**直接改了**这个 userscript(仓库源 + vault 安装副本),被用户当场亮灯纠正:这是**开源仓库的功能增强**,应走 handoff 由 feishukanban CC 实施(铁律 #2,**非紧急功能不走跨边界例外**;之前 5-26/5-27 例外都是紧急 bug)。OB CC 已把两个 `.js` **回退到干净 v1**,改由本 handoff 移交。

## 目标 / 非目标

**目标**:v1 的「状态 + 描述」之后,加 2 个**可选** prompt:用时(数字 → `act=`)、完成度(suggester → `done=`)。产出行:`- 日期 | 状态 | review=… / act=1.5 / done=标准完成`(空字段不写)。

**非目标**:
- 不动 `sync.py`(`act→actual_hours` / `done→completion` 已支持)。
- 不动 QuickAdd 注册(OB 侧已注册 `log-today-detail-choice`,vault `data.json` 已就绪,re-install 不影响它)。
- 不加 `plan=` / `est=`(本次用户只要 用时 + 完成度)。

## 接口契约(`sync.py` 已支持,无需改)

`_DETAIL_KEY_ALIASES`:`act → actual_hours`(float)、`done → completion`(text)、`review → review`(text)。
完成度合法值(对齐 task 模板注释):`最小完成 / 标准完成 / 超额完成 / 阻碍 / 未启动`。

```js
    // Step 3.5:用时小时数(可选,数字 → act=)
    const actRaw = await quickAddApi.inputPrompt(
      "⏱️ 今日用时几小时?(可选,如 1.5,Enter 跳过)",

    const COMPLETION_OPTIONS = ["标准完成", "最小完成", "超额完成", "阻碍", "未启动"];
    const completionPick = await quickAddApi.suggester(
      ["⏭️ 跳过(不记完成度)", ...COMPLETION_OPTIONS],
      ["", ...COMPLETION_OPTIONS]
    );
```

**2. `newLine` 构造改为多 key 拼接**(替换原来的单 `review=` 逻辑):

```js
    const keyParts = [];
    if (description && description.trim()) keyParts.push(`review=${description.trim()}`);
    if (actRaw && actRaw.trim() && !isNaN(parseFloat(actRaw.trim()))) keyParts.push(`act=${parseFloat(actRaw.trim())}`);
    if (completionPick && completionPick.trim()) keyParts.push(`done=${completionPick.trim()}`);
    let newLine = `- ${today} | ${statusPick}`;
    if (keyParts.length) newLine += ` | ${keyParts.join(" / ")}`;
```
