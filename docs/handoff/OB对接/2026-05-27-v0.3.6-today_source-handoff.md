# Handoff:v0.3.6 计划/非计划语义重构 — OB 端 journal dataview 改查询条件

## 一句话需求

仓库侧已上线 `today_source` 字段(planned / unplanned / 空),把"计划/非计划"语义从误用的 `priority=P3` 中**完全剥离**。OB 端需要改 journal 模板 + 历史 journal 的 dataview 查询条件,用 `today_source` 替代 `priority` 区分计划/非计划段。

## 背景:为什么这是 ADHD 自觉察的关键

| 旧设计(错位) | 新设计(语义正交) |
|---|---|
| `priority=P3` 同时表达"价值低" + "非计划" | `priority` 仅表达**任务价值/紧急度** |
| 真正"价值高但临时插入"的 task 显示在 P0-P2 计划段 → 错位 | `today_source=unplanned` 独立标记"临时插入" |
| 真正"价值低但提前规划"的 task 显示在 P3 非计划段 → 错位 | `today_source=planned` 独立标记"早晨规划" |

**ADHD 复盘价值**:看历史 journal,知道**那天**有多少 task 是规划好的、多少是临时插入的。这是工作随机性 vs 计划性的真实数据。

### 件 ① journal 模板 dataview 改查询条件(必做)

**文件**:`/Users/aim5/Documents/OB/03 Resources/素材库/模版/日志模版 5.0 1.md`

找到「🎯 今日计划」段下的 dataview block,大概长这样(v0.3.4 修复后):

```dataview
TASK
FROM "04 Inbox/task"
WHERE !contains(file.name, "_说明")
  AND contains(today_history, this.file.day)
  AND (priority = "P0" OR priority = "P1" OR priority = "P2")
SORT priority ASC, created DESC
```

**改为**:

```dataview
TASK
FROM "04 Inbox/task"
WHERE !contains(file.name, "_说明")
  AND contains(today_history, this.file.day)
  AND (today_source = "planned" OR today_source = null)
SORT priority ASC, created DESC
```

同样,「🐿️ 今日非计划」段下的 dataview block,大概长这样:

```dataview
TASK
FROM "04 Inbox/task"
WHERE !contains(file.name, "_说明")
  AND contains(today_history, this.file.day)
  AND priority = "P3"
SORT created DESC
```

**改为**:

```dataview
TASK
FROM "04 Inbox/task"
WHERE !contains(file.name, "_说明")
  AND contains(today_history, this.file.day)
  AND today_source = "unplanned"
SORT created DESC
```

**关键变化**:
- 计划段:`today_source = "planned" OR today_source = null`(null 兜底,让历史 task 默认归"计划")
- 非计划段:`today_source = "unplanned"`(严格匹配,排除 null)
- **priority 不再参与计划/非计划区分**,只用于排序
