# skill 约束补强 — 反向回执

## 结论

已采纳 handoff 的两条核心约束:

1. **执行明细同日只能一条 state record**:新增 sync.py lint + strict flag,并在共享 SKILL.md 里写成硬约束。
2. **交付段 vault 外文件必须绝对路径**:新增 sync.py lint + strict flag,并在共享 SKILL.md 里把相对路径反例独立突出。

## 已落地

### sync.py

- 新增 `lint_execution_detail_same_day()`:
  - 扫「## 📈 执行明细」原始 bullet,不仅看解析后的 detail dict。
  - 能抓 `2026-06-11 | ...` 多条,也能抓 `2026-06-11 早/上午 | ...` 这种原本不会被解析的行。
  - 默认 warning;带 `--strict-detail-one-per-day` 时拒推。
- 新增 `lint_delivery_paths()`:
  - 扫「## 📦 交付」里的行内 code、markdown link、OB 双链、裸文本路径。
  - 对 `docs/...`、`src/...`、`.claude/...`、`CHANGELOG.md` 等明显相对路径 warning。
  - 默认 warning;带 `--strict-delivery-paths` 时拒推。
- 新增 CLI flags:
  - `--strict-detail-one-per-day`
  - `--strict-delivery-paths`
- 已接入 `--task-md`、`--create-task` dry-run/apply、`--push-all-today` 单条循环。
