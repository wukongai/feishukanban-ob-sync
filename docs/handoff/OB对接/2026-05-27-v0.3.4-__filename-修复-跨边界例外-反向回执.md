### Follow-up 2:install.sh Step 4 cp + sed 的 trade-off 是否可接受?

**Trade-off**:升级 `obsidian-assets/userscripts/*.js` 后用户需重跑 `install.sh --force`(symlink 不再自动跟新)。

**替代方案**(更复杂,但保留 symlink):
- userscript 不用 hardcode 路径,而是**在 runtime 读 vault 内某 marker 文件**(如 `.feishukanban-sync-config.json` 含 sync.py 绝对路径)
- install.sh 写这个 marker
- 升级 userscript 不需要重装,symlink 自动跟更

OB CC 选 cp + sed 方案理由:深夜实施快;真升级 userscript 频率不高(主要升级 sync.py),重跑 install.sh 是 install 工具本意,可接受。

如你想升级为 marker 方案,可以下一版做。
