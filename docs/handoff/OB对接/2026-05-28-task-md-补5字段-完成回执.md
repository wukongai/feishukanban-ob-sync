### 代码层面亮点

#### 1. `parent_task` 自关联 link 字段双向解析(新)

**Forward**(OB→飞书):
- 新 helper `resolve_parent_task_record_id(parent_task_name, vault_root)`(sync.py 1373 附近)
- `[[2026-05-20-XXX]]` → vault `04 Inbox/task/2026-05-20-XXX.md`(直接路径优先 + rglob fallback) → 读父 task frontmatter `feishu_record` → 写入飞书 link 字段 `[<rec_id>]`(数组形态)
- 父 task 无 `feishu_record`(未 sync) / vault 找不到 → warning + 跳过该字段(**不阻断其他 4 字段**)

**Reverse**(飞书→OB):
- `_extract_fields_from_feishu_row(row, fields_meta, config, ob_index=...)` 加 `ob_index` 可选参数
- 飞书 link 字段返回 `[{"id": "rec...", ...}]` → 反查 `_scan_ob_task_md_by_feishu_record` 索引 → 取 path.stem → 拼 `[[<stem>]]`
- ob_index 无对应 rec_id → 留空字符串(不污染 frontmatter)

#### 2. 反向同步正文 H2 段(首次实现)

新 4 个函数:

```python
_REVERSE_SYNC_H2_WHITELIST = [
    ("delivery", "## 📦 交付", "交付"),
    ("user_story", "## 👥 用户故事", "用户故事"),
]

def _read_h2_section_content(file_path, h2_pattern) -> str: ...
def update_h2_section_in_task_md(file_path, h2_title, new_content) -> bool: ...
def _diff_h2_sections_with_feishu(p, fs_fields) -> tuple[dict, list]: ...
def _apply_h2_updates(rid, p) -> int: ...
```

- 段存在 → 正则替换段内容(保留 H2 标题行)
- 段不存在 → 在「## ✅ 完成标记」之前插入完整新 H2 段(**老 task md 兼容关键**)
- 防御策略对齐 `PRESERVE_OB_IF_FS_EMPTY`:**飞书侧空 → 保留 OB**(避免误清用户本地内容)
- 找不到「## ✅ 完成标记」标识 → 放弃插入(防御不规范 task md)

#### 3. `pull_today_from_feishu` 三分支统一接 H2 同步

- `_compute_field_diff` 同时计算 frontmatter diff + H2 diff
- dry-run 输出加 H2 段 diff 显示:`📑 交付: (空) → 交付物: / - [...] / - [...]`(多行 `\n` → ` / ` 压缩 + 截 50 字)
- 三个 apply 分支(plan_set_true / plan_skip / plan_set_false)都在 `update_md_frontmatter` 后调 `_apply_h2_updates` 同步 H2 段
- plan_missing 自动建 task md 时传 `ob_index`(让新 task md 的 parent_task 能反向找 wikilink)

#### 4. `_create_task_md_from_feishu_record` 模板扩展

模板加 `quality / actual_hours / parent_task` frontmatter 行 + `## 📦 交付 / ## 👥 用户故事` H2 段(空就空)。

## 偏离 spec 之处

无明显偏离。仅做了几点**实现细节决策**:

1. **`_extract_fields_from_feishu_row` 多接收一个 `ob_index` 可选参数**(handoff 没明说但需要):用于 parent_task 反向 record_id → OB wikilink 解析。无 ob_index 时 parent_task 字段留空(不写注释),原 handoff 提的"写注释`# parent_task: <rec_id>`"方案被弃用 — 因为 frontmatter 写 `# ...` 不被 YAML 解析为有效值,会引起 OB dataview 误解析,留空更稳。
2. **`parent_task` 加入 `PRESERVE_OB_IF_FS_EMPTY` 集合**:跟 quality / actual_hours 一起,防御性"飞书空 → 保留 OB"。理由是 parent_task 是建模 task 拆分关系,飞书侧不该轻易清空覆盖 OB。
3. **dry-run 输出 H2 段 diff 用 ` / ` 替代 `\n` + 截 50 字**:多行内容在终端展开会刷屏,折成单行更易审。完整内容 apply 时仍写入。
